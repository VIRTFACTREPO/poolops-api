## [Heading1] 10. Web Dashboard — Visual Direction
This section defines the visual character of the web dashboard specifically. It is distinct from the mobile app specs above. The reference aesthetic is the Velix AI Agent dashboard — a mobile-first SaaS UI with a refined, near-white neomorphic card language. That aesthetic has been adapted here for the PoolOps desktop dashboard context.

## [Heading2] Aesthetic Direction
Refined operational minimalism. The dashboard should feel like a precision instrument — calm, uncluttered, and immediate. Every element earns its place. Density is managed through spacing and hierarchy, not visual noise.
The light mode variant of this dashboard (as opposed to the default dark mode spec in section 2) should feel almost physical — cards that sit softly on the surface, not floating aggressively above it. This is the neomorphic quality observed in the Velix reference: same-family colours between card and background, with only a whisper of shadow to establish depth.

## [Heading2] Core Visual Principles

## [Heading3] 1. Near-white palette
Page background sits at #F5F7FA — not pure white, not grey. Cards use #FFFFFF with a shadow so soft it reads as material depth rather than elevation. The result is a surface that feels cohesive and physical.

#F5F7FA
Card background
#FFFFFF

0 2px 8px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04)

## [Heading3] 2. Soft neomorphic cards
Cards do not use heavy borders or dramatic shadows. Instead, a very light, diffused shadow (see above) creates the impression of slight elevation from the same background material. This is deliberately subtle — the card and background are nearly the same colour family, differentiated only by surface sheen.
Active or selected cards add a subtle coloured border (#0EA5E9 at 30% opacity) to indicate state without aggression.

## [Heading3] 3. Muted accent states
Connected/active states use soft green fills (#F0FDF4 background, #22C55E text). Action/connect states use soft lavender (#EFF6FF background, #0EA5E9 text). These are pill-shaped with gentle border radii (20px+). They should feel like status indicators, not buttons shouting for attention.

## [Heading3] 4. Category filter tabs
Horizontal filter tabs (e.g. All | Active | Inactive | Plan) use an underline-style active indicator rather than filled pills. Active tab: Pool Blue text + 2px bottom border. Inactive: #9CA3AF text. No background fill on tabs.

## [Heading3] 5. Status badge taxonomy
Status badges follow a consistent pill language across all surfaces. They are small (12px font, 3px 8px padding) and always pair colour with text. They never rely on colour alone.
•  Popular / Flagged: soft red (#FEF2F2 bg / #DC2626 text / #FECACA border)
•  New / Active: soft green (#F0FDF4 bg / #16A34A text / #BBF7D0 border)
•  Premium / Warning: soft amber (#FFFBEB bg / #D97706 text / #FDE68A border)
•  Neutral / Inactive: soft grey (#F9FAFB bg / #6B7280 text / #E5E7EB border)


## [Heading3] 6. Bottom / side navigation
The active nav item uses a dark pill (near-black #111827 background, white text/icon). Inactive items are icon-only or icon+label in #9CA3AF. The active pill creates a strong but contained focal point — it anchors the user's position without dominating the layout.
On desktop, this pattern translates to the left sidebar: active item gets a dark pill background rather than the default blue tint, maintaining the same visual language across surfaces.

## [Heading3] 7. Typography hierarchy
Section titles are large and confident (H1 at 28px/700). Body content is compact (14px/400). The contrast between these two creates clear scanning hierarchy without needing heavy decoration. Inter remains the typeface throughout — its neutrality is a feature, not a limitation, in an operational tool.

## [Heading3] 8. Stat/metric display
Key metrics (job counts, API connections, instance counts) are displayed as large numerals (32px+/700) beneath small uppercase labels (11px/500). These are grouped in a horizontal strip near the top of the dashboard. Mini bar charts or subtle waveform graphics can accompany each stat to add visual rhythm without conveying precise data.

## [Heading3] 9. Agent / item cards
Two-column card grids are the primary layout unit for listing items (agents, integrations, jobs). Each card contains: an icon (40px, rounded square), a category badge (top-right), a title (15px/600), a description (13px/400/#6B7280), and a status/action pill at the bottom. Cards have 16px internal padding and 12px gap between them.

## [Heading3] 10. Motion principles
Interactions should feel responsive but not theatrical. Hover states on cards transition border colour smoothly (150ms ease). Status pills fade between states (200ms ease). Page transitions are instant — no full-page animations. Loading states use a subtle pulse on the card skeleton, not a spinner.

## [Heading2] Light Mode vs Dark Mode
The mobile app uses the light mode palette throughout (sections 2–9 above). The web dashboard defaults to dark mode (section 2, Dark Mode subsection) for long operational use. However, the visual direction in this section (section 10) describes the light mode character that informs all design decisions — including how dark mode components should feel: the same softness and restraint, applied to dark surfaces.
If a light mode toggle is added to the web dashboard in a future version, it should implement the Velix-inspired near-white palette described in this section directly.

## [Heading2] What to Avoid
•  Heavy drop shadows that make cards feel like they're floating 20px above the page
•  Filled/solid colour tab bars — they compete with content
•  Gradients on primary buttons or backgrounds — this is not a marketing page
•  Bright, saturated accent colours used at full opacity for large areas
•  Dense icon-only navigation without clear active state labelling
•  Card borders that are the same weight as content dividers — establish hierarchy
